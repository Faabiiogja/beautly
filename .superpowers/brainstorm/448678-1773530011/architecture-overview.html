Arquitetura Geral — Beautly MVP

Next.js Monolito · App Router · Vercel · Supabase

── Request Flow ──────────────────────────────────────
Browser (celular/web) → nomedaestetica.beautly.com
Vercel Edge Network → middleware.ts (identifica tenant pelo host)
Next.js App Router → roteia para booking ou admin
Server Components / API → Supabase PostgreSQL

Princípios aplicados

Shared DB
PostgreSQL único com tenant_id em todas as tabelas. Row-Level Security via Supabase garante isolamento.
Edge Middleware
Resolve tenant pelo host antes de qualquer Server Component rodar. Zero roundtrip extra.
Server Components
Dados carregados no servidor. Cliente recebe HTML pronto. Mobile sente na velocidade.
